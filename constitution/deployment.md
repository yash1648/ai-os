# Deployment Rules

Rules governing deployment policy, release process, and production access.

## Release Process

- **Rule:** No diff generated by a worker may be deployed to production without passing through the full Review Pipeline (reviewer + architecture guardian gates).

- **Rule:** Release tags must follow semantic versioning (`vMAJOR.MINOR.PATCH`). Each release must include a corresponding entry in the changelog.

- **Rule:** All releases must be built from the `main` branch. Feature branches and worker-generated diffs must merge through a pull request.

## Production Access

- **Rule:** Direct production database writes from worker-generated code are forbidden. All data mutations must flow through the kernel's API layer.

- **Rule:** Production Prometheus metrics endpoints must be protected by an authentication proxy. The `/metrics` endpoint must not be publicly accessible.

- **Rule:** The kernel must log all configuration changes at startup to the audit log, including the active constitution version and enabled feature flags.
